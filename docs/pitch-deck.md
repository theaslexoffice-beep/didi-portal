## Slide 2: The Problem

---

### Citizens Are Voiceless in the Bureaucracy

**The Reality:**
- **No effective channel** for citizens to get grievances resolved
- Bureaucracy is **slow, disconnected, and unresponsive**
- Common citizens don't know **where to go** or **whom to approach**
- Paper complaints get **lost in files** — no tracking, no accountability
- **No feedback loop** — citizens never know what happened to their complaint
- Vulnerable populations (**elderly, rural, illiterate**) are completely excluded

---

**The Cost:**
- Erosion of trust between citizens and government
- Unresolved issues fester into larger social problems
- Democratic participation remains a slogan, not reality

---

## Slide 3: The Vision

---

### Technology + AI + Community = Change

**We envision an India where:**
- Every citizen has a **voice** that reaches the right ears
- Bureaucratic walls become **transparent windows**
- Technology bridges the **trust deficit** between people and governance
- AI amplifies human empathy, not replaces it
- Community participation ensures **no one is left behind**

---

**The DIDI Approach:**
> *Make grievance redressal **accessible**, **accountable**, and **effective** — one complaint at a time.*

## Slide 4: How It Works

---

### The DIDI Journey — From Voice to Action

```
┌─────────────────────────────────────────────────────────────┐
│  STEP 1: Submit Complaint                                   │
│  📱 WhatsApp-first approach (Voice/Text/Image)              │
│  🌐 Web portal backup for smartphone users                  │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  STEP 2: AI Categorisation & Prioritisation                 │
│  🤖 GPT-4o-mini powered triage                              │
│  🏷️ Auto-categorisation (water, sanitation, roads, etc.)   │
│  🔺 Maslow-pyramid prioritisation (urgent cases first)       │
│                                                             │
├─────────────────────────────────────────────────────────────┤
│  STEP 3: Admin Moderation                                   │
│  👥 Human review for sensitive/urgent cases                 │
│  ✅ Validation before formal escalation                     │
├─────────────────────────────────────────────────────────────┤
│  STEP 4: Formal Escalation to Authorities                   │
│  📄 Government-formatted complaint generation                 │
│  🎯 Direct routing to relevant department/official            │
├─────────────────────────────────────────────────────────────┤
│  STEP 5: Track Resolution                                   │
│  📊 Real-time status updates to complainant                 │
│  ⏱️ SLA monitoring and deadline alerts                     │
├─────────────────────────────────────────────────────────────┤
│  STEP 6: Social Amplification (if needed)                   │
│  📢 Unresolved cases highlighted on social media              │
│  📰 Media engagement for public-interest issues               │
└─────────────────────────────────────────────────────────────┘
```

## Slide 7: Technology Stack

---

### Modern, Scalable, Cost-Effective

| Component | Technology | Purpose |
|-----------|------------|---------|
| **Frontend** | Next.js 14 (React) | Fast, SEO-friendly web portal; PWA support |
| **Backend & Database** | Supabase (PostgreSQL) | Real-time updates, authentication, storage |
| **Messaging** | WhatsApp Cloud API | Primary citizen interface; rich media support |
| **AI/ML** | GPT-4o-mini (OpenAI) | Complaint categorisation, sentiment analysis, summary generation |
| **Triage Engine** | Custom Python + Pinecone | Priority scoring; duplicate detection; trend analysis |
| **Hosting** | Vercel + Supabase Cloud | Serverless, auto-scaling, minimal DevOps overhead |
| **Monitoring** | Vercel Analytics + Logflare | Uptime tracking, error monitoring, usage analytics |

---

**Data Security:**
- End-to-end encryption for citizen communications
- GDPR-aligned data handling practices
- No personal data sold or shared with third parties
- Regular security audits and penetration testing
